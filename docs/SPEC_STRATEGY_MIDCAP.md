# 規格:選股策略轉向——中型流動性股 × 動能 × 買氣 ×（即時）題材

> 狀態:**草案,待使用者確認決策點後實作**。起因:使用者檢視決策軌跡後判斷
> 「多方幾乎都被採納、都買到阿呆股」,並定調要「少而精、有新聞有基本面、不易被
> 主力操控」。本規格用實際資料把方向從「純大型權值」微調為**「中型流動性甜蜜點」**,
> 並誠實標出資料缺口與可回測邊界。
> 原則沿用前幾份規格:**可觀測先於優化、驗證先於學習、誠實標註無法回測的部分**。

## 0. 問題診斷(2026-07-14 實錘,非印象)

現行系統 = **全市場動能篩選器 + 一層薄辯論**,三個結構性病:

1. **選股專挑「沒新聞的極端飆股」**:用 RS20 全市場前段動能,最近 8 檔候選近 30 日
   新聞**全部 0 則**(擎亞/磐亞/華容…),因為 RS20 撈的是漲最兇的微型投機股,而新聞
   媒體不報這種股。→ 辯論的新聞分析師結構性沒料,多空只剩技術+籌碼。
2. **候選池沒有品質/流動性門檻**:唯一過濾是「量≥500張」。全市場 1948 檔裡,近一日
   成交金額 <5千萬 的有 **1175 檔**——這區塊籌碼薄、主力好拉抬,RS20=1.00 常常就是
   「主力拉上去」,你追進去、隔天跳空、停損。
3. **實際成績證實**:46 筆已平倉 AI 部位,平均 +0.19% 毛報酬(扣費是負的)、勝率 50%、
   典型小賺大賠。「都買到不好的股票」是資料證實的事實。

## 1. 三個關鍵問題的答案(決定策略邊界)

### Q1:大型股為什麼不適合純動能?
動能溢酬來自「趨勢延續尚未被完全定價」的時間差。大型權值股(台積電/鴻海)法人覆蓋
密、資訊透明、流動性極高,趨勢**早被反映在價**——你 RS20 看到它強時通常已是共識,
追高邊際報酬低。這是學術界的 **size × momentum 交互**:動能溢酬在小型顯著高於大型。
所以純大型股+動能=報酬縮水。**但「大型/中型股+題材+基本面」是不同邏輯**,不靠時間差,
靠基本面轉折/題材發酵,大中型反而合適。

### Q2:為什麼新聞/題材無法回測?
- **可回測**:價、量、法人買賣超、月營收、技術指標——都有逐日時間序列,能「回到某天
  只用當時已知資料」重現。
- **不可回測=新聞/題材**:(a) 我們的 `market_signals` 新聞 `created_at` 全部集中在
  2026-06-30 之後,往前沒有逐日新聞存檔;(b) 更本質的:「當時市場怎麼解讀這則新聞」
  是當下的 sentiment,事後補不出來(同一則擴廠新聞多頭是利多、空頭沒人理)。這是所有
  新聞驅動策略的通病。→ **新聞那塊只能靠即時 LLM 判斷,無法用歷史證明,誠實面對。**

### Q3:中型股能同時有新聞+買氣+動能嗎?——能,而且是甜蜜點
實際資料(2026-07-14):
- **新聞覆蓋不是只有大型股**:近 30 日有新聞的個股,價格級距 17(>500)/**39(100-500)/
  41(30-100)**/19(<30)——**中間段最密**,不是大型股。
- **「有新聞 且 投信近期常買」交集 = 61 檔**——現成、可運作的候選池。
- 你現在推薦 0 新聞,不是「中型股沒新聞」,是**RS20 前段篩選專挑那些沒新聞的極端值**。

**中型流動性股是三條件交集點**:有媒體覆蓋(辯論有料)、有法人買氣(可回測)、有動能
(仍有溢酬、可回測)、市值/流動性夠大(抗操控)、且比純大型股更有動能空間、比微型飆股
更可回測。

## 2. 新候選池設計(可落地,已對過現有資料)

### 2.1 硬門檻(全市場 → 可交易的中型流動性池)
| 門檻 | 值(可調) | 資料來源 | 目的 |
|---|---|---|---|
| **成交金額下限** | 近 5 日均額 ≥ **2 億/日** | daily_prices(close×volume) | **抗操控核心**:排除 1175 檔低流動性阿呆股,剩 ~466 檔(>10億206+2-10億260) |
| 收盤價 | ≥ 15 元 | daily_prices | 排除雞蛋水餃股 |
| RSI(14) | 45 ~ 88 | technical_indicators | 排除超弱/極端超買 |
| 排除警示 | (待補,見資料缺口) | — | 處置股/全額交割 |

> **為什麼用成交金額不用市值**:我們沒有市值資料,但操控的本質是「低流動性、籌碼薄、
> 好拉抬」,成交金額直接命中這個風險——比市值更貼近「抗操控」目標。市值大但每日量薄的
> 股票一樣危險,成交金額抓得到。

### 2.2 評分(在池內排序,取前 N 進辯論)——可回測部分
沿用現有 `score_candidates` 因子但重新配權,核心從「極端動能」轉為「中度動能 × 買氣 × 成長」:
```
趨勢(中度,不追極端):
  相對強度 RS20(但排名限縮在池內,不是全市場前段) + 多頭排列持續 + 60日動能
買氣(可回測,題材的量化代理):
  投信連續買超(天數×量體) + 外資買超 + 三大法人合計
成長(可回測基本面):
  月營收年增 + 年增加速(>20%)
```
🔶 **決策點1**:動能權重要不要調降?現行趨勢版動能權重最高(抓飆股),新方向要不要把
動能降權、把「買氣+成長」升權,讓選股偏向「有基本面撐的中度強勢股」而非「最兇的飆股」?

### 2.3 題材濾網(即時,不進回測)
辯論前為每檔候選檢索近 30 日相關新聞(已有 `_recent_news_map`)。因為候選改成中型股,
這層**終於會有料**(不像現在 0 則)。🔶 **決策點2**:新聞要當「硬門檻」(近 30 日無任何
新聞就不選)還是「軟加分」(有新聞的辯論時多一份題材證據)?硬門檻更「少而精」但可能
過度限縮;軟加分較彈性。

## 3. 辯論層要不要一起改(回應「薄辯論」)

現行:20 檔一次掃、單輪、多空各寫一次不互看。TradingAgents 是**單股深入、多輪來回**。
兩條路:
- **A(小改)**:維持結構,但候選縮到 ~8 檔(池已縮小+成交金額門檻),讓每檔辯論更有料,
  且新聞真的接進去。成本低。
- **B(大改)**:改成「先粗選 8 檔 → 逐檔多輪辯論(空方回應多方、多方再駁)」。更接近
  TradingAgents,品質更高但 LLM 呼叫數翻倍(額度壓力,見下)。

🔶 **決策點3**:先做 A(小改、低成本)看效果,還是直接上 B?

> ⚠️ **額度現實**:Gemini 免費版**每日每專案 20 次呼叫**。現行每天已用 bull+bear+judge=3 次
> +新聞摘要等。B 方案逐檔多輪會爆量。若要認真做多輪辯論,可能需要付費 key 或換模型。
> 這是規格必須正視的硬限制。

## 4. 資料缺口(誠實清單,有些要先補才能做)

| 缺口 | 現況 | 影響 | 補法 |
|---|---|---|---|
| **市值/股數** | 無 | 用成交金額代理即可,不阻擋 | (選配)FinMind/TWSE 抓股數 |
| **完整財報** | `financials` 表**空的** | 「有基本面」目前只能用月營收 | 抓 MOPS 財報填 financials |
| **處置/警示股清單** | 無 | 無法精準排除全額交割股 | TWSE 每日公布,可爬(成交金額門檻已擋掉大部分) |
| **歷史新聞** | 2026-06-30 後才有 | 新聞層無法回測(如 Q2) | 無解,接受即時 LLM 判斷 |

## 5. 回測與驗證計畫

- **可回測主體**:成交金額門檻 + 動能/買氣/成長評分 + E4 趨勢騎乘出場——這套完全可用
  歷史資料跑,和現行趨勢版同框架,只是候選池與權重不同。
- **對照**:新中型版 vs 現行趨勢版 vs 0050,用既有 `run_backtest` 的風險調整後指標
  (Sharpe/回撤/Calmar)比較。預期:**報酬上限降低(不追極端飆股),但回撤/波動改善、
  且更接近可實盤複製**(流動性夠、不會買到拉不動或賣不掉的股票)。
- **新聞層**:標註為「即時加分,無法回測」,不納入回測績效宣稱。

## 6. 實作順序(建議)

```
1. 候選池改造：成交金額門檻 + 池內評分重配權（stock_selector）——可回測
2. 回測對照：中型版 vs 趨勢版 vs 0050（風險調整後）——用數據決定權重
3. 定案後接上即時辯論（新聞這次有料）+ 已修好的 guardrail
4. （選配，看效果）辯論改多輪 B 方案 / 補財報 / 補處置股清單
```
