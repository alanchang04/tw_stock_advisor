## 2. 推理層改造設計

### 2.1 餵料升級(讓多方「有東西可論證」)
`format_candidates_for_llm` 每檔加入(全部已算好,只是沒給):
```
趨勢:RS20 全市場百分位 / 多頭排列連續N日 / 60日動能
基本面:月營收YoY
籌碼:投信連買N日(含量體) / 法人買賣超(修正為張)
題材:近30日相關新聞標題(C3 關鍵字檢索,有才附)
```
多方 prompt 改為:每檔論點須含**趨勢結構/籌碼/題材至少兩類證據**,禁止只複述當日
漲幅與 RSI;並要求「這檔最強的反對理由是什麼、為何仍值得」(預防性自辯)。

### 2.2 結構化辯論契約(JSON,取代散文)
```json
bull: {"picks":[{"stock_id","thesis","evidence_fields":["rs20","rev_yoy",...],"preempt_rebuttal"}]}
bear: {"vetoes":[{"stock_id","reason","severity":"veto|caution"}],"market_concerns":[...]}
```
- severity 分級:`veto`(絕對不該碰)/`caution`(風險提示)。
- 好處:裁決可被**程式**逐檔比對;C1 引用驗證直接驗 `evidence_fields`;決策軌跡頁可
  渲染成對決表(多方論點 vs 空方回擊,一檔一列)。

### 2.3 裁決問責(核心——不可無腦採納)
1. 裁決輸出每檔**必附** `objections_addressed`:空方對此檔的每條異議 → `接受(剔除/降級)`
   或 `駁回+具體理由`(理由必須引用資料欄位,受 C1 驗證)。
2. **程式 guardrail(非 LLM,決定性)**:被標 `veto` 的股票若仍入選且無非空駁回理由 →
   **自動剔除、遞補第6名**,並在決策軌跡標紅「裁決違規被系統攔截」。
   🔶 決策點1:veto 的強度——(a) 軟:有書面駁回即可入選(預設);(b) 硬:veto 一律剔除,
   LLM 不可推翻。
3. 裁決 prompt 明示:「照抄多方全名單且未逐條回應空方=無效輸出,將被重試」。

### 2.4 反駁輪(可選,+1 LLM 呼叫)
空方對多方的 JSON 逐檔回擊一輪,裁決看三份文件。更接近真辯論,成本 +1 呼叫、
+30~60 秒。🔶 決策點2:要不要開(可先關,量測 2.3 效果後再議)。

### 2.5 與主規格的銜接
- **C1 引用驗證**:結構化 JSON 讓「每個數字對回輸入欄位」容易十倍——建議與本規格同捆實作。
- **C2 穩定性量測**:改造前先跑一輪基線(現況 judge 重跑3次的重疊率),改造後再跑一輪
  對比——本規格的驗收證據之一。
- **Phase B 驗證器**:P0-3 是其第一條規則的提前實作。

## 3. 量測與驗收(過程指標,誠實聲明:LLM 層無法回測報酬)

| 指標 | 現況(2026-07-13 實測) | 目標 |
|---|---|---|
| 空方 veto 被裁決明確回應的比例 | 0%(全部只貼 risk_note) | 100%(程式 guardrail 保證) |
| 裁決照抄多方全名單的比例 | 2 run 中 1 次(50%) | 有 veto 時 0% |
| 多方論點含趨勢/題材證據的比例 | ~0%(全是當日漲幅+RSI) | ≥80%(結構化欄位可程式統計) |
| 引用數字與輸入不符(幻覺)率 | 未量測 | C1 上線後 <5% |
| 餵料完整性 | 法人 1000×錯誤+race 全 0 | quality_gate 擋下+標註 |

驗收方式:改造上線後一週,人工抽看 3 個 run 的決策軌跡頁:多方論點不再是資料表複述、
每個 veto 都有明確裁決回應、guardrail 攔截有記錄。
